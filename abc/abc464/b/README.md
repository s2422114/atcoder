# ABC464 B - Crop

- URL: https://atcoder.jp/contests/abc464/tasks/abc464_b
- 結果: AC（ヒントあり。自力では方針が出なかった）
- 解法: 削除するのではなく、黒(#)が存在する最小の長方形の範囲を求めて、その中だけ出力
  - 全マス走査し、# を見つけるたびに min/max で4つの境界を更新
  - min_up/min_left は大きい値で初期化、max_down/max_right は0で初期化
- 詰まった点:
  - 問題文が「白い行を削除する」なので、erase で消す方法ばかり考えていた
  - min/max で領域を絞る発想がまったく出てこなかった
  - 実装ミス3つ: if の {} 忘れ / 条件が '.' （白）になっていた / 出力ループが < だった
- 学び:
  - 【重要】問題文の操作をそのまま実装するとは限らない。
    「削除する」→「残すものを選ぶ」に言い換えると簡単になることが多い
    （ABC459 の「X文字目を削除」でも同じ変換が出てきた）
  - グリッドで「〜が存在する範囲」を求める = min/max で境界を更新する（バウンディングボックス）
  - if に {} を付けないと直後の1文しか対象にならない → 必ず {} を付ける
- 復習: 要（自力で方針が出なかった）
