# ABC465 B - Parking

- URL: https://atcoder.jp/contests/abc465/tasks/abc465_b
- 結果: AC（WA → 場合分けの漏れを修正）
- 解法: 駐車時間[A,B]と高い時間帯[L,R]の重なりを場合分けで計算
- 詰まった点:
  - 最初、区間がまったく重ならないケース（B <= L / R <= A）の分岐を最後に書いたが、
    条件が緩い上位の分岐に吸われて到達しなかった
  - 例: L=9,R=17,A=1,B=5 のとき `a <= l && b <= r` が true になり、
    time_LR = b - l = -4 と負の時間になっていた
- 学び:
  - 場合分けの条件が緩いと、意図しないケースが紛れ込む。
    「a <= l && b <= r」だけでは重なりの有無を判定できていなかった
  - 対策1: 特殊ケース（重ならない）を一番最初に弾く
    if (b <= l || r <= a) { 全部が安い時間 }
  - 対策2: 各分岐の条件を厳密に書く（l < b を足して重なりを保証）
- 別解: 場合分けせず重なりを直接計算する
    int high = min(B, R) - max(A, L);
    if (high < 0) high = 0;
    int low = (B - A) - high;
  区間の重なりは max(左端同士) と min(右端同士) で求まる（頻出テクニック）
